# 心愿 Moss 精灵 · 产品建议

这份文件用于记录产品设计上的取舍建议，主页面 `index.html` 已经按产品说明书方式呈现。

## 1. 推荐采用分体式本地架构

最推荐版本：

- 玩偶端：ESP32-S3 + 麦克风 + 喇叭 + 灯带 + 触摸 + 电池。
- 电脑端：Ollama/llama.cpp + Qwen/Hermes + Agent + 记忆库 + 控制客户端。
- 可选增强：Luckfox Pico Pro 做 Linux 网关、缓存和调试服务。

这样可以在 1000 元以内完成，同时保留“本地 AI 毛绒终端”的核心卖点。

## 2. 毛绒尺寸建议放宽到 18cm

12cm 可以做，但内部空间会很紧。

18cm 的优势：

- 喇叭声音更好。
- 电池更容易固定。
- 线材不容易挤压。
- 充电口、开关和维护口更好隐藏。
- 散热和安全余量更大。

如果目标是展示效果和长期稳定，建议优先选 18cm 左右可拆内胆玩偶。

## 3. 加一个实体静音开关

这是很实用的小设计：

- 用户不想被监听时可以直接关闭麦克风。
- 调试时更安全。
- 产品感更强。

推荐位置：

- 隐藏在背部拉链内。
- 或做成尾巴/挂件式开关。

## 4. 增加离线安静模式

电脑不在线时，玩偶不需要完全“死机”。

离线状态可保留：

- 摸头亮灯。
- 低电量提示。
- 固定提示音。
- 呼吸灯待机。
- 长按进入休眠。

这样即使 AI 大脑不在线，Moss 精灵仍然像一只活着的小伙伴。

## 5. 记忆要可视化

长期记忆是卖点，但也必须可控。

建议电脑端提供一个本地网页：

- 查看 Moss 记住了什么。
- 删除某条记忆。
- 导出全部记忆。
- 清空设备。
- 查看心愿种子成长记录。

这会让产品更可信，也更适合长期使用。

## 6. 电脑控制默认保守

第一版只默认开放安全指令：

- 调音量。
- 播放/暂停。
- 打开网页。
- 打开指定应用。
- 截图。

默认关闭：

- 删除文件。
- 发送消息。
- 关机/重启。
- 支付相关操作。
- 修改系统设置。

高权限能力可以做，但必须二次确认。

## 7. 技术替换要留接口

建议把以下模块都做成可替换：

- 大模型：Qwen / Hermes / Llama。
- Agent：OpenClaw / LangGraph / 自研状态机。
- 语音识别：Whisper.cpp / faster-whisper。
- 语音合成：Piper / Edge TTS / CosyVoice。
- 通讯协议：WebSocket / MQTT / HTTP。

这样后面升级不会推翻整套产品。
