# 心愿 Moss 精灵 · 本地 AI 毛绒终端产品说明书

版本：v1.1  
定位：个人专属、本地优先、可触摸的 AI 智能体毛绒终端

## 1. 产品概述

心愿 Moss 精灵是一只可以手拿、可以摸头、会听你说话、会用灯光表达情绪、能记住心愿并控制电脑的本地 AI 毛绒终端。

它不是普通联网玩具。推荐采用“毛绒交互终端 + 本地电脑 AI 大脑”的分体式架构：

- 玩偶端负责语音采集、语音播放、触摸反馈、RGB 情绪灯、电量提示和局域网通讯。
- 本地电脑端负责大模型对话、语音识别、语音合成、长期记忆、心愿种子、智能体调度和电脑控制。
- Luckfox Pico Pro 作为可选 Linux 网关，用于设备桥接、缓存、调试和后续扩展。

一句话说明：

> 把本地电脑上的 AI 大脑装进一只可爱的毛绒终端里，让它像桌面小伙伴一样陪伴、记忆和帮你操作电脑。

## 2. 产品定位

- 形态：12-18cm 毛绒玩偶，推荐可拆内胆，便于维护。
- 场景：电脑桌面陪伴、心愿记录、语音助理、局域网电脑控制。
- 人设：温柔、可爱、记忆力好、懂得陪伴，但对危险指令谨慎。
- 隐私：语音、记忆和控制指令默认在本地网络内处理。
- 成本：基础版 450-650 元，增强版 700-900 元，目标控制在 1000 元以内。

## 3. 工作原理

### 3.1 毛绒交互终端

负责用户能直接感受到的部分：

- 麦克风采集语音。
- 喇叭播放回复。
- 触摸模块识别摸头、轻拍、长按。
- WS2812 RGB 灯显示情绪、聆听、思考、低电量和错误状态。
- ESP32-S3 与本地电脑保持局域网通讯。

### 3.2 边缘控制层

负责硬件控制和设备桥接：

- ESP32-S3 管理音频、触摸、灯效和 Wi-Fi 心跳。
- Luckfox Pico Pro 可选，用于局域网桥接、缓存、设备管理和调试页面。
- Luckfox 不作为 7B/8B 大模型主机，避免性能、内存和发热问题。

### 3.3 本地电脑大脑

负责 AI 和自动化能力：

- 运行 Qwen / Hermes 等本地大模型。
- 运行 Whisper.cpp / faster-whisper 做语音识别。
- 运行 Piper / Edge TTS 缓存音色 / CosyVoice 做语音合成。
- 运行 OpenClaw 或自研 Agent 做任务调度。
- 管理 SQLite 记忆库、向量索引、心愿种子和审计日志。
- 执行电脑控制，例如打开应用、搜索、截图、快捷键、输入文字。

- 电池、电源线和功放必须重点做绝缘与固定。

## 5. AI 技术选型

### 5.1 大模型

推荐：

- 中文陪伴与工具调用：Qwen3 4B/8B Q4，或 Qwen2.5 7B Instruct Q4。
- 角色风格与 Agent：Hermes 3 8B GGUF。
- 低配置电脑：Qwen3 1.7B/4B 或更小模型。

运行方式：

- 简单部署：Ollama。
- 性能可控：llama.cpp。
- Apple Silicon：可考虑 MLX。

### 5.2 智能体

推荐设计一个 `agent_adapter` 适配层：

- 如果 OpenClaw 部署稳定，用 OpenClaw 做记忆、工具路由和任务调度。
- 如果 OpenClaw 暂时不好接入，用 FastAPI + 自研状态机先实现。
- 玩偶端协议不要绑定某一个 Agent 框架，后续可以替换。

### 5.3 语音系统

- 语音识别：Whisper.cpp / faster-whisper，建议在电脑端运行。
- 语音合成：Piper 或 Edge TTS 缓存音色，后续可升级 CosyVoice / GPT-SoVITS。
- 唤醒方式：第一版建议“摸头唤醒 + 语音唤醒词”，比纯语音唤醒更稳定。

## 6. 核心功能

### 6.1 AI 自然聊天

- 中文闲聊。
- 情绪安抚。
- 日常提醒。
- 轻量知识问答。
- 根据记忆生成更贴近用户的回复。

### 6.2 长期记忆

- 记住用户名字、偏好、常用软件、常用指令和重要心愿。
- 支持查看、搜索、删除、导出。
- 敏感记忆可单独加密。

### 6.3 情绪灯效

- 待机：柔和绿色呼吸。
- 聆听：青绿色渐亮。
- 思考：蓝绿流动。
- 开心：暖黄色闪烁。
- 拒绝危险指令：橙红短闪。
- 低电量：红色慢闪。

### 6.4 触摸互动

- 摸头：问候、撒娇、进入聆听。
- 长按：静音或休眠。
- 轻拍：确认、打断或“我在”反馈。

### 6.5 电脑控制

第一版建议支持：

- 打开应用。
- 调整音量。
- 播放/暂停音乐。
- 打开网页。
- 搜索内容。
- 截图。
- 输入文字。
- 执行快捷键。

高风险操作必须二次确认：

- 关机/重启。
- 删除文件。
- 发送消息。
- 付款相关动作。
- 修改系统设置。

## 7. 心愿种子系统

心愿种子是 Moss 精灵的核心差异化功能。

流程：

1. 用户说出一个心愿。
2. Moss 精灵把它保存成一颗“种子”。
3. 用户每次完成相关行动，种子获得成长值。
4. 种子从“种子、发芽、长叶、开花”逐步成长。
5. 心愿完成后生成一份成长记录。

字段设计：

```json
{
  "wish_id": "wish_001",
  "title": "学会画一只 Moss 精灵",
  "reason": "想做自己的角色设定",
  "stage": "sprout",
  "energy": 42,
  "next_action": "今晚画 20 分钟草图",
  "memory_links": ["喜欢青绿色", "第一次说想画画"]
}
```

## 8. 安全与隐私

权限分级：

- L0：只聊天，不控制电脑。
- L1：安全控制，例如音量、播放暂停、打开网页。
- L2：编辑控制，例如输入文字、快捷键、打开应用。
- L3：危险控制，例如关机、删除文件、发送消息，必须二次确认。

必须具备：

- 本地保存，不默认上传云端。
- 记忆可查询、可删除、可导出。
- 电脑控制行为写入审计日志。
- 控制令牌可撤销。
- 玩偶丢失时可以禁用设备或清空记忆。

## 9. 使用流程

1. 用户摸头或说出唤醒词。
2. 玩偶灯光进入聆听状态。
3. 玩偶采集语音并发送到本地电脑。
4. 本地电脑完成语音识别、记忆检索和模型推理。
5. Agent 判断是否需要调用工具或控制电脑。
6. 玩偶播放回复并同步灯效。
7. 重要信息写入本地记忆或心愿系统。

## 10. 实施建议

### 阶段一：电脑端原型

- 跑通本地模型。
- 实现本地记忆库。
- 实现 5 个安全电脑控制指令。
- 做一个简单 Web 控制台。

### 阶段二：裸板联调

- ESP32-S3 跑通 WS2812 灯效。
- 触摸模块触发事件。
- 麦克风和喇叭完成录放音链路。
- ESP32-S3 与电脑端 WebSocket / MQTT 通讯。

### 阶段三：毛绒集成

- 确定玩偶尺寸。
- 固定电池、主板、喇叭、麦克风和灯带。
- 隐藏开关和充电口。
- 验证散热、续航和安全。

### 阶段四：展示样机

- 连续演示 30 分钟。
- 完成聊天、记忆召回、心愿种子、电脑控制和危险指令拒绝。
- 优化音色、灯效和人设。

## 11. 好建议

- 建议把毛绒尺寸从 12cm 放宽到 18cm，项目成功率会明显提高。
- 建议第一版保留一个实体静音开关，比纯软件静音更安心。
- 建议加入“离线安静模式”：电脑不在线时，玩偶仍能做摸头反馈、呼吸灯和固定提示音。
- 建议把记忆管理做成可视化页面，用户能看见 Moss 记住了什么。
- 建议危险电脑控制默认关闭，等基础体验稳定后再逐步开放。
