# 阶段一验收清单

## 已完成

- 本地 Web 控制台：`http://127.0.0.1:8787`
- SQLite 数据库：`data/moss_stage1.db`
- 语音对话台：
  - 点击说话，浏览器把语音转成文字
  - 页面展示“我听到”的转写文本
  - 页面展示本轮用户输入和 Moss 回复
  - 支持浏览器本地朗读回复，不需要 MiniMax Key
- 本地记忆库：新增、查询、删除、导出
- 自动记忆：
  - “记住...”
  - “我喜欢...”
  - “我不喜欢...”
  - “我叫/我是...”
- 本轮记忆命中：每次回复展示参考了哪些旧记忆
- 会话记录：每轮聊天写入本地 SQLite
- 心愿种子：创建、成长值推进、阶段变化、完成状态
- 聊天接口：优先 MiniMax，其次 Ollama，失败时本地规则回复
- 模型配置：支持 `.env` / 环境变量，并显示 MiniMax/Ollama 可用状态
- MiniMax TTS：
  - 查看 TTS 配置状态
  - 临时密钥请求，不写入源码、数据库或日志
  - 支持模型、音色、速度、音量、音调调节
  - 生成 mp3 并通过本地播放器预览
- 5 个安全控制指令：
  - 打开网页
  - 搜索内容
  - 打开白名单应用
  - 调整系统音量
  - 保存屏幕截图
- 安全机制：
  - 控制指令执行前需要控制台确认
  - URL 限制为 `http/https`
  - 应用启动使用白名单
  - 控制指令写入审计日志
  - MiniMax 密钥只从环境变量或本页临时请求读取，不写源码和数据库

## 当前限制

- 还没有 ESP32-S3 通讯协议，阶段二再接 WebSocket/MQTT。
- 没有 MiniMax Key 且没有本机 Ollama 时，聊天会使用本地 fallback 逻辑。
- 浏览器语音输入依赖 Web Speech API；如果浏览器不支持，可先用文字输入测试。
- MiniMax TTS 已接入，但真实合成依赖网络和 API 额度。
- 截图指令可能捕获屏幕隐私内容，所以放在 L2 并要求确认。

## 下一步建议

1. 你确认阶段一语音对话体验是否符合“电脑端先模拟娃娃”的预期。
2. 在页面临时填 MiniMax Key 测试真实大模型回复，或安装/启动 Ollama。
3. 阶段二开始定义 ESP32-S3 事件协议：
   - `touch_event`
   - `audio_chunk`
   - `battery_status`
   - `light_effect`
   - `speak_text`
